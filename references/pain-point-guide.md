## 一、评价来源优先级

| 平台 | 优势 | 劣势 |
|------|------|------|
| **TapTap** | 中文玩家为主，评论详细具体 | 需要API或网页抓取 |
| **App Store** | 评分权威，覆盖面广 | 评论量相对少 |
| **Google Play** | 多语言，适合出海游戏 | 国内参考价值低 |
| **Steam** | 评测机制成熟，正负分离 | 需要登录 |

**实操建议：** 优先TapTap，中文评价信息密度最高。

### 按严重程度分级

| 级别 | 信号 | 应对 |
|------|------|------|
| 🔴 P0 | 大量集中出现，1星差评超30% | 必须解决才能降低流失率 |

## 四、策划洞察提取方法

### 从抱怨中找"机会词"

玩家抱怨 = 需求未被满足的信号：

| 玩家抱怨（负向） | 真实需求（正向） | 设计机会 |
|----------------|---------------|--------|
| "体力不够用" | 想多玩一会儿 | 设计替代体力消耗的趣味循环 |
| "无内容可做" | 想要更多内容 | 设计UGC/常驻活动 |
| "太肝了" | 想轻松玩游戏 | 降低重复劳动，增加休闲玩法 |
| "抽卡太贵" | 想要更多免费奖励 | 设计抽卡保底+免费获取渠道 |
| "匹配不到人" | 想和朋友一起玩 | 降低组队门槛，跨服匹配 |

## 六、工具使用说明

### fetch_reviews.js 参数说明

```bash
node fetch_reviews.js <游戏名> [平台]
# 示例
node fetch_reviews.js 原神 appstore  # 仅AppStore
node fetch_reviews.js 原神 taptap    # 仅TapTap
```

- `/workspace/tmp/pain_point_raw.json` — 原始评价数据
- `/workspace/tmp/pain_point_report.md` — 生成的完整报告

## 七、常见游戏分析案例

### 二次元抽卡游戏常见痛点
1. 体力限制严格（长草期问题）
2. 抽卡概率不透明
3. 角色强度膨胀（旧角色淘汰）
4. 剧情跳过功能缺失
5. 刷刷刷重复劳动

### 竞技游戏常见痛点
1. 服务器稳定性（卡顿/掉线）
2. 匹配机制（ELO/胜率控制）
3. 外挂问题
4. 平衡性更新（削弱争议）
5. 社交功能缺失

### SLG游戏常见痛点
1. 零氪玩家被碾压
2. 赛季末期鬼服
3. 离线收益设计争议
4. 肝度极高
5. 付费差距过大
